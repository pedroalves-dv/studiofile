Three Approaches for the Sticky Hero

How the TOTEM title stays in view while image columns scroll past it

Recommended
A

Absolute columns + sticky in flow

Section has an explicit tall height (300dvh on md+). Image columns are position: absolute — purely decorative. The TOTEM wrapper is in normal flow, sticky top-0 h-dvh, so it fills the viewport and sticks for the entire scroll duration.

<section class="relative md:h-[300dvh]"> <!-- Absolute columns (decorative) --> <div class="absolute inset-0 hidden md:flex"> <Col stagger={0} /> <Col stagger="50dvh" /> <Col stagger="100dvh" xl-only /> </div> <!-- Sticky content — in flow --> <div class="sticky top-0 h-dvh flex items-center justify-center z-10"> <HeroContent /> </div> </section>
CSS-only No JS Simple Performant
B

Columns in flow, sticky overlay

Columns are in normal document flow (not absolute), so they naturally drive the section height. TOTEM is in a separate absolutely-positioned layer, with a sticky child inside.

<section class="relative"> <!-- Columns drive height (in flow) --> <div class="hidden md:grid md:grid-cols-2"> <Col /> <Col /> </div> <!-- Absolute overlay with sticky child --> <div class="absolute inset-0"> <div class="sticky top-0 h-dvh flex items-center justify-center"> <HeroContent /> </div> </div> </section>
Organic height sticky in absolute ⚠ CSS gotcha
C

JS scroll-pinning (GSAP pin)

GSAP ScrollTrigger pins the TOTEM while the section scrolls. Gives fine-grained control over enter and exit animations, but contradicts the CSS-first philosophy and the project's existing use of GSAP (scroll-only, removed).

<section ref={sectionRef}> <Columns /> <div ref={totemRef} class="absolute"> <HeroContent /> </div> </section> // useEffect: ScrollTrigger.create({ trigger: sectionRef, pin: totemRef, start: "top top", end: "bottom bottom", });
JS-heavy GSAP already removed Against CSS-first rule

Why Approach A wins

The sticky mechanic

  • sticky top-0 h-dvh fills one full viewport
  • Sticks for section-height − 100dvh = 200dvh of scroll
  • TOTEM is always centred via flex items-center justify-center
  • Un-sticks cleanly as ProductSpotlight enters

The stagger mechanic

  • Col 1: no offset — 3 full images from top
  • Col 2: pt-[50dvh] — starts half-viewport lower
  • Col 3: pt-[100dvh] (xl only) — starts 1 viewport lower
  • Section uses overflow-hidden — bottom clipped naturally
